FSV CLI is a command-line tool that offers the most features from UI for scanning folders, creating reports, and exporting them with simple commands. It primarily consists of these commands - pr [args] for permission report, fr [args] for forder report, or [args] for owner report, sr [args] for share report, and ur [args] for user permission report. FSV CLI exports the reports to Excel, HTML, or CSV files.

fsv

Use fsv --help to list all commands.

> fsv --help
FolderSecurityViewer CLI (fsv), Version: 2.3.0.0 - Enterprise Edition

Usage: fsv [options] [command]

Options:
  -?|-h|--help  Show help information

Commands:
  fr  Start folder report
  or  Start owner report
  pr  Start permission report
  sr  Start share report
  ur  Start user permissions report

Use "fsv [command] --help" for more information about a command.

fsv pr

Use fsv pr [path] to start permission report on given directory.

> fsv pr [arguments] [options]
Arguments:
    [path] A path of directory to scan. Use quotes if the path has whitespace in it.
Options:
    -h|-?|--help Show help information
    -e|--export <fileType> [Optional] Export to file - excel, csv, or html. Default is html
    -p|--path <fileName> [Optional] An export file path.
    -d|--db [Optional] Export to database.
    -df|--difference [Optional] Compare sub-directories for difference in permissions, and export to default directory.
    -dp|--difference-path <fileName> [Optional] An export file path for difference of permissions.

Description

fsv pr [path] starts permission report to analyze NTFS permissions of given folder in your Windows server environment, and exports the result in specified file format.

  • [path] - A local or UNC folder path on which the NTFS permissions analysis has to be performed. This argument is required.
  • -e or --export - Exports to one of supported file formats - Excel, HTML, or CSV. If this option is omitted, HTML is used.
  • -p or --path - Use this option to export permission report result to path of your choice. Default location of exported file is FolderSecurityViewer\Export in your Documents folder.
  • -d or --db - Use this option to save permission report result in database.
  • -df or --difference - Use this option to scan the difference in permissions of sub-folders, and exports the file in default location. If omitted, the scan doesn't initiate.
  • -dp or --difference-path - Use this option to export difference of permissions report result to path of your choice. Default location of exported file is FolderSecurityViewer\Export in your Documents folder.

fsv fr

Use fsv fr [path] to start folder report on given directory.

> fsv fr [arguments] [options]
Arguments:
    [path]  A path of directory to scan. Use quotes if the path has whitespace in it.
Options:
    -h|-?|--help  Show help information
    -e|--export <fileType> Export to file - excel, csv, or html. Default is HTML
    -p|--path <fileName> An export file path.

Description

fsv fr [path] starts folder report to flatten specific folder, and its subfolders into a single list with details like owner, size of folder, and number of files in it.

  • [path] - A local path of folder. This argument is required.
  • -e or --export - Exports to one of supported file formats - Excel, HTML, or CSV. If this option is omitted, HTML is used.
  • -p or --path - Use this option to export folder report result to path of your choice. Default location of exported file is FolderSecurityViewer\Export in your Documents folder.

fsv or

Use fsv or [path] [owner] to start owner report on given directory.

> fsv or [arguments] [options]
Arguments:
  [path]   A path of directory to scan. Use quotes if the path has whitespace in it.
  [owner]  A name of owner
Options:
  -h|-?|--help  Show help information
  -e|--export <fileType>  Export to file - excel, csv, or html. Default is HTML
  -p|--path <fileName>    An export file path.

Description

fsv or [path] starts owner report to scan specific folder, and its subfolders to find out the folders in which specified user is set as an owner.

  • [path] - A local path of folder where the scan is started. This argument is required.
  • [owner] - Name of a user to scan the folder for its ownership. If this argument is not provided, the export file lists all folders with name of their owners.
  • -e or --export - Exports to one of supported file formats - Excel, HTML, or CSV. If this option is omitted, HTML is used.
  • -p or --path - Use this option to export folder report result to path of your choice. Default location of exported file is FolderSecurityViewer\Export in your Documents folder.

fsv ur

Use fsv ur [path] [user] to scan given Active Directory user deep in directory hierarchy.

> fsv pr [arguments] [options]
Arguments:
    [path] A path of directory to scan. Use quotes if the path has whitespace in it.
    [user] A name of Active Directory user.
Options:
    -h|-?|--help Show help information
    -e|--export <fileType> [Optional] Export to file - excel, csv, or html. Default is html
    -p|--path <fileName> [Optional] An export file path.

Description

fsv pr [path] starts permission report to analyze NTFS permissions of given folder in your Windows server environment, and exports the result in specified file format.

  • [path] - A local or UNC folder path on which the NTFS permissions analysis has to be performed. This argument is required.
  • [user] - A name of Active Directory user to search in given directory or its sub-directories. This argument is required.
  • -e or --export - Exports to one of supported file formats - Excel, HTML, or CSV. If this option is omitted, HTML is used.
  • -p or --path - Use this option to export permission report result to path of your choice. Default location of exported file is FolderSecurityViewer\Export in your Documents folder.
